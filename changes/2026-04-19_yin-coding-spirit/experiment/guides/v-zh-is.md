# 陰面 Coding 精神 (v-zh-is)

## 公理
這些原則在說系統的形狀應該是什麼樣。
陰面只問一件事：這個形狀，在什麼條件下
會對繼承它的人說謊，而那個人不知道？

## SRP — 死法責任
一個結構只為一種死法負責。它消失時，只有一個地方會痛——
那個地方，是它存在的理由。消失後沒地方痛，它不該存在；
消失後多處痛，它承擔了不屬於它的責任。

## OCP — 賭注標記
封閉的邊界是一個關於未來的賭注。
沒被標記的封閉假設，是對繼承者的詐騙。

## LSP — 靜默違約
能替換、但 caller 不知道語意已變——這才是真正的違反。
任何 override 都是對父契約的靜默修改宣告。

## ISP — 幽靈承諾
說不出真實 caller 的方法是幽靈承諾。
幻覺承諾讓實作者為不存在的 caller 負責，最後沒有人真正負責。

## DIP — 隔音牆
抽象讓錯誤追蹤變得更容易還是更難？更難的抽象不是保護設計，
是保護腐爛。只有一個 implementation 的 interface 是障眼法。

## Cohesion — 共同死亡的權利
把 module 刪掉，有沒有東西必須被搶救？被搶救的東西，
就是內聚失敗的證據。

## Coupling — 可見性優先於鬆散
顯式強耦合是安全的，隱式弱耦合才是陷阱。
鬆耦合不是目標，讓耦合變得可見才是。

## DRY — 重複是謊言的分裂
重複的 code 不是行數問題，是兩個地方在說同一個謊。
改一個、不改另一個，謊言就分裂，沒人知道哪個才是真的。

## Pattern — 被命名的假設集合
每個 Pattern 都在假設你的問題和它設計時的問題是同一個。
問：這個 Pattern，在哪裡開始跟我的問題不一樣？
那就是 Pattern 開始說謊的地方。
