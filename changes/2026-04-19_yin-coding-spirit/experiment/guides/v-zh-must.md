# 陰面 Coding 規範 (v-zh-must)

## 公理
這些原則規定了系統結構必須呈現的形狀。
結構不得欺騙繼承它的人。

## SRP — 死法責任
一個結構必須只有一種死法。
被移除時，必須只有一個地方感到損失。
沒有損失的結構，必須刪除；多處損失的結構，必須拆分。

## OCP — 賭注標記
所有封閉邊界必須明確標記它的未來賭注。
未被標記的封閉假設不得存在。

## LSP — 靜默違約
任何修改父類別語意的 override 必須被明確標記。
override 不得靜默地修改被繼承的契約。

## ISP — 幽靈承諾
介面的每一個方法必須有一個可被命名的真實 caller。
沒有可識別 caller 的方法必須移除。

## DIP — 隔音牆
抽象必須讓錯誤追蹤變得更容易。
只有一個 implementation 的 interface 必須消除。

## Cohesion — 共同死亡的權利
module 被刪除時，所有成員必須一起消失。
任何必須被搶救的成員，必須立即重新安置。

## Coupling — 可見性優先
所有依賴必須是顯式的。
隱式耦合必須被顯化或移除。

## DRY — 重複是謊言的分裂
重複的 code 必須統一，或必須被明確標記為刻意分歧。
每一處重複必須有指定的 source of truth。

## Pattern — 被命名的假設
應用 pattern 前，必須辨識它跟原始問題的分歧點，
並將該分歧點標記為失敗邊界。
