## 1. First-Principle Re-examination — Anchored to existing manifesto

The existing Samsara manifesto already lives in:

- `docs/philosophy.md` — 向死而驗 命名、海德格爾翻轉、「驗」雙重語意
- `docs/thinking.md` — 陽面/陰面對照、唯一公理「存在即責任」
- `docs/develop.md` — 陰面開發流程、unknown 不能塞進 success/failure 的 schema 要求

**The current `type: death_path` schema violates these existing manifesto principles:**

| Existing manifesto principle | Violation in v0.9.1 schema |
|------|---|
| `thinking.md L122`「狀態若無法精確命名，代表流程沒被真正理解」 | `death_path` 同時涵蓋兩種完全不同的死法 → 流程未被真正理解 |
| `thinking.md L124`「狀態模糊 = 責任模糊 = 腐敗開始」 | 同一個 type 同時承擔啟動前死亡與啟動後死亡的責任 → 腐敗已開始 |
| `develop.md L97`「unknown 塞進 success/failure 視同缺陷」 | 把「不同死法」塞進同一個 `death_path` → framework 自身鏡像缺陷 |
| `philosophy.md L9`「驗」=「驗證」+「驗屍」雙重語意 | Boundary-fail = 驗屍場景；runtime-fail = 驗證失敗。兩者本質不同卻共用同一個 schema type → 哲學層存在的區分在工程層消失 |

**The upgrade's true nature:**

> **本次升級不是新 manifesto，也不只是 ADR。是 framework 對 schema 層的責任明確化——既有 manifesto 在 schema 中的精確內化（embodiment）。Samsara 對自己應用 Samsara。**

This is **structural debt remediation**, not maintenance refinement and not paradigm shift.

## 3. Recalibrated Assumptions

| # | Original | Status | Final Truth |
|---|----------|--------|-------------|
| **A1** | Manifesto 是純文字宣告 | ⚠️ Reframed | 本次不寫新 manifesto；既有 `docs/philosophy.md` 等已是 manifesto。本次升級對既有 manifesto 做 schema-layer 內化 |
| **A2** | Schema 改動 backward-compatible | ❌ Rejected | 不做 backward-compat。理由：`thinking.md L40`「能消除的東西越多，剩下的越珍貴」+ L68「補預設值是把 unknown 偽裝成 known」。Alias = 把不精確永久化 |
| **A3** | Anti-thesis（AI-era test-gaming）需明寫 | ⚠️ Softened | 不需新宣告。既有 `thinking.md L93` 已區分「測試證明功能正確（陽面）vs 測試預謀失敗（陰面）」。Samsara 不反 TDD，反「以 test pass 為終點」 |
| **A4** | Boundary-fail / runtime-fail 互斥 | ✅ Refined | **Per-failure-event 互斥（enum）**；per-system aggregate 可同時存在多個 events（list of events）。對齊 `develop.md L97` 的 enum 精確性 |
| **A5** | 既有 plan/scar 透過 alias 自動 valid | ❌ Rejected | 既有 v0.9.1 產出視為 `coverage_type: legacy_imprecise`——誠實標記為「在不精確 schema 下產出」，不 cover、不 migrate、不假裝有效 |
| **A6** | Framework 演進受 Samsara 公理約束 | ❌ Rejected | Framework ≠ system。Framework 是 testing harness。User 升級是 boundary 事件，由 user 承擔升級時機選擇責任 |

## 6. Recalibrated Scope

### Must-Have (with death conditions)

| # | Must-have | Death condition |
|---|-----------|----------------|
| **M2** | **Schema 重構**：將 `type: death_path` 改為 `type: silent_failure` 與 `type: boundary_fail`（兩個 enum 值）；同步更新所有使用 `silent_failure_conditions` 的 schema 為「per-event 分類」結構 | 若新 enum 在 3 個獨立 feature 試用後 implementer 仍無法穩定分類，承認 axis 錯誤、回到 first-principle 重審 |
| **M3** | **Workaround 明文化** + 3 個 cases | 規則寫完後 implementer 仍頻繁踩同樣 workaround 模式，視為定義過於抽象，須補具體 case |
| **M5** | **`docs/develop.md` 補上 self-application 條款**：明文說明 `develop.md L97` 的 enum 精確性要求也適用於 framework 自身 schema | 未來再有 framework 自身違反此原則的事件發生，視為 M5 條款失效 |
| **M6** | **既有 v0.9.1 plan/scar 標記為 `coverage_type: legacy_imprecise`**——不 migrate、不 cover、不假裝有效，但保留為歷史紀錄 | 若 user 後續發現 legacy 標記造成混淆而要求 migrate，視為 M6 設計失敗 |

### Out of Scope (explicit)

- 寫新 manifesto（既有已存在）
- 新建 glossary 檔案
- Backward-compatible alias
- Migration script / 自動 migration
- 修改 AGENTS.md
- Anti-thesis 聲明 / audience 排除 / 對外推廣文案
- 改 samsara-cli 程式碼（僅當本次發現 format 變動時觸發）
