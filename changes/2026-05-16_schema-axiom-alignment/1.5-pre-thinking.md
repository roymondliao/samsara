## III. Uncertainties（我無法判斷 Research 想要哪邊）

### U1: 對 framework 而言「system」是誰？death test 該測什麼？

**雙端可能性**：

| 端 | 解讀 | Death test 樣態 |
|----|------|---------------|
| **(a) Framework prose + schema 為 system** | implementer 行為是 user input | 對 framework 文檔 grep/lint：舊詞彙是否漏網、glossary cross-ref 是否完整、deprecated alias 是否有 warning |
| **(b) Implementer 行為為 system** | framework 是 environment | 跑 ≥3 個 implementer case study 驗證能否正確分類 |

**我的傾向**：(a)——framework 的 boundary 是文檔自身的一致性。Case study (b) 屬於 ship 之後 monitoring 範疇。

**使用者回應**：[ ] 同意 (a) / [ ] 應為 (b) / [ ] 兩者皆需，定義組合：_______________
